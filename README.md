# HZIP - 基于 Huffman 编码的压缩工具

## 简介
HZIP 是一个用 C 语言实现的简易文件压缩/解压工具，基于 Huffman 编码算法。  
支持将文本文件（`.txt`）压缩为 `.hzip` 格式，并可从 `.hzip` 还原为原始文本文件。

## 编译方法
确保系统已安装 GCC 编译器，然后在项目根目录下执行：

```bash
make
```

编译完成后会生成可执行文件 `hzip`（Linux/macOS）或 `hzip.exe`（Windows MinGW）。

## 使用方法
### 压缩
```bash
./hzip input.txt
```
- 输入文件扩展名必须为 `.txt`。
- 压缩后输出文件为 `input.hzip`，与输入文件在同一目录。

### 解压
```bash
./hzip -u input.hzip
```
- 解压时需使用 `-u` 选项，且输入文件扩展名必须为 `.hzip`。
- 解压后输出文件为 `input.txt`。

## 功能限制
该工具为课程作业演示，存在以下已知限制：

1. **仅支持 ASCII 文本文件**  
   字符频率表固定为 128 个 ASCII 字符，无法处理中文、UTF-8 等多字节编码。

2. **文件大小上限**  
   使用 `int` / `long` 存储大小，理论上限约 2GB。超大文件可能溢出。

3. **强制包含 NUL 字符**  
   为保证解压终止，程序在统计频率时为 `NUL`（ASCII 0）添加一次计数。若原文件不含 NUL，解压时会多输出一个空字符。**不要用于压缩二进制文件**。

4. **扩展名硬编码**  
   压缩输入必须为 `.txt`，解压输入必须为 `.hzip`，否则报错退出。

5. **无错误恢复机制**  
   若压缩文件损坏或格式不符，解压可能崩溃或输出错误内容。

6. **编码长度上限**  
   单个字符的 Huffman 编码长度不得超过 `MAXSIZE`（默认 32）。极端权重分布下可能截断（触发程序报错退出）。

7. **压缩率**  
   仅对文本文件有效，对已压缩数据或随机数据可能反而增大体积。压缩率仅供参考。

## 已知问题与未来改进
- **工程化**：当前代码采用上下文结构体传递，但部分错误处理仍不完善（如解压时未使用文件末尾大小信息）。
- **压缩格式**：未记录原始文件大小和尾部位数，解压依赖 NUL 字符停止，可能导致多余输出。
- **跨平台**：未处理 Windows 与 Linux 换行符差异。
