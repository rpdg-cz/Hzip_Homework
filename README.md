# HZIP - 简易 Huffman 压缩工具

## 简介
基于 Huffman 编码实现的文本文件压缩/解压工具。支持将 `.txt` 文件压缩为 `.hzip` 格式，并可还原。

## 编译方法
```bash
gcc -o hzip.exe hzip.c huffman_impl.c -I.
```
注意：请确保 `huffman_impl.c` 中 `#include "Huffman.h"` 且文件在同一目录。

## 使用方法
### 压缩
```bash
hzip.exe input.txt
```
输出：`input.hzip`

### 解压
```bash
hzip.exe -u input.hzip
```
输出：`input.txt`

## 功能限制

1. **仅支持 ASCII 文本文件**  
   字符频率表固定为 128 个 ASCII 字符，无法处理中文字符或其他编码（如 UTF-8）。

2. **文件大小限制**  
   使用 `int` 类型存储文件大小和压缩后大小，理论上限约 2GB，实际受 `ftell` 返回值限制。对于超大文件可能溢出。

3. **特殊字符处理**  
   为保证解压正确，在统计频率时强制为 `NUL` (ASCII 0) 字符添加一次出现次数。因此，若原文件中不含 NUL，压缩后会多出一个 NUL 编码，导致解压后多一个空字符。**不推荐压缩含 NUL 的二进制文件**。

4. **文件扩展名依赖**  
   - 压缩输入必须为 `.txt` 扩展名（程序会检查）。  
   - 解压输入必须为 `.hzip` 扩展名。  
   若文件未按此命名，程序会报错退出。

5. **无错误恢复机制**  
   如果压缩文件损坏或格式不符，解压过程可能崩溃或产生错误输出。

6. **内存使用**  
   - Huffman 树节点数最多约 256 个，内存占用很小。  
   - 编码表使用固定 `MAXSIZE=32`，若某个字符编码长度超过 31 位（理论上在 128 个叶子时最大深度可达 127），会截断。实际上由于权重分布，编码长度通常不超过 32，但仍存在风险。

7. **压缩率**  
   仅对文本文件有效，对已压缩或随机数据可能反而增大体积。压缩率输出仅供参考。

## 已知问题

- 代码工程化不足：`hzip.c` 直接 `#include "huffman_impl.c"`，应分离编译。
- 全局变量 `ctex` 为单例，无法同时处理多文件。
- 文件操作缺少 `fseek` 错误检查。
- 压缩时未写入文件尾部有效位数信息，解压时依靠 `NUL` 字符结束，可能造成最后几个字节误读。
